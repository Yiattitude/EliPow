# AGENTS.md

## 1. 项目名称
**电荔——上电AI学习助手**

## 2. 项目定位
本项目是一个面向**电气相关专业本科生**的**个性化学习成长系统**。  
平台由电力行业专属知识图谱驱动，以多智能体协同学习引擎为核心，通过“技术领域-岗位标准-知识点-应用场景-操作步骤”五级分层网络，以及个人动态图谱演化，自动完成从学情识别到个性化学习方案输出的完整闭环，实现“在校所教所学即为企业所用”。

## 3. 项目目标
- 帮助电气相关专业本科生建立清晰的专业学习路线，转向以学生为中心的主动学习模式
- 结合通用五级分层知识图谱与个人动态知识图谱，精准识别知识缺口
- 利用多智能体协同（总控、检索、推理、更新），生成高度定制化、可解释的学习路径
- 引入产教融合：将企业真实运维案例与现行标准转化为教学资源，并利用 AI 反向赋能高校教师优化大纲
- 提供基于 GraphRAG 的专业知识关联推理问答
- 构建基于多智能体调度闭环的：**测评 → 智能推理推荐 → 学习行为追踪 → 个人图谱演化 → 路径纠偏**

## 4. 核心用户
- 电气相关专业本科生
- 主要面向上海电力大学学生场景
- 高校教研团队（受惠于产教融合缺口分析）

## 5. 项目范围边界
### 5.1 做的内容
- 用户注册、登录、个人知识图谱初始化
- 基于 GraphRAG 的多智能体关联推理与学习路径推荐
- 五级分层知识网络的构建与维护
- 产教融合资源库对接与大纲缺口分析
- 强关联可溯源的专业问答
- 动态学习进度记录与图谱自进化

### 5.2 不做或后期再做的内容
- 社交/泛水区社区
- 课程交易/付费内容
- 复杂的人工多角色审批系统
- 泛人群教育平台

## 6. 核心产品理念
### 6.1 个性化与主动学习是主线
系统的核心是：
- 通过诊断测试初始化个人图谱
- 推理智能体结合岗位目标生成专属学习路径
- 伴随学习行为实时演化个人图谱权重，告别“一刀切”

### 6.2 问答要具备关联推理能力 (GraphRAG)
从单纯的“文档检索”跃迁至“知识关联推理”。AI 回答需要穿透知识图谱的依赖链路，给出精准且必须附带来源引用的解答。

### 6.3 产教融合资源赋能
不局限于书本，推送企业一线运维案例和技术标准。反向为教师提供课程大纲与岗位要求的偏差报告。

## 7. 技术栈
### 7.1 前端
- React + Tailwind CSS + shadcn/ui + Lucide Icons
- ECharts / AntV G6 (用于复杂知识图谱与个人图谱可视化)

### 7.2 后端
- Spring Boot + Spring Security
- MySQL (用户、业务核心数据)
- **Neo4j** (图数据库，存储五级分层知识网络、个人关系与 GraphRAG 支撑)
- Redis

### 7.3 AI 智能体
本项目的 AI 核心采用多智能体协同架构，基于 **Spring AI** (并适度引入或自研 Agent 编排机制)：
- `ChatClient` / `Advisor` / `PromptTemplate`
- RAG (全面升级为基于 Neo4j 的 **GraphRAG**)
- 多智能体分工调度与 Tool Calling

### 7.4 架构分工
- **Java 后端**：负责业务系统、图谱节点维护、权限、数据库事务管理
- **AI 智能体引擎**：基于多 Agent（总控、检索、推理、更新）进行复杂的逻辑推演与图谱穿透
- **前端**：负责多维度的图谱可视化、学习路径展示和交互

## 8. AI 架构要求 (重构为多智能体协同)
### 8.1 推荐方案
采用 **“多智能体协同学习引擎 + GraphRAG + 图数据库”** 的架构。

### 8.2 智能体职责分工
- **总控调度智能体**：接收用户请求，识别学情，调度其他专属智能体，合并结果。
- **知识检索智能体**：依托 Neo4j 数据库执行复杂的 Cypher 查询，提取“岗位-知识点-场景”关联子图。
- **推理路径生成智能体**：核心大脑，比对个人图谱（当前掌握度）与岗位能力要求，自动推演出最佳学习序列路径。
- **知识库更新智能体**：监控企业真实案例和标准，将其结构化并自动更新入库。

### 8.3 GraphRAG 作用
- 提升问答中的深层因果/关联逻辑准确率。
- 使资源推荐不仅仅基于文本相似度，而是基于严密的“前置知识/包含关系”。

## 9. 智能体使用约定
### 9.1 输出要求
- AI 输出必须高度结构化。
- 各智能体之间的通信需包含清晰的上下文（如个人当前能力指标）。
- 推理结果（如学习路径）必须具备“可解释性”（即：为什么推荐这个知识点，它属于哪个岗位标准的哪一环）。

## 10. 功能模块
### 10.1 个人图谱与画像
- 从传统的静态标签升级为与通用图谱映射的“个人动态图谱”。
- 输出详细的薄弱节点链路及目标能力缺口。

### 10.2 专业能力测评模块
结合操作步骤级任务，精准校准个人图谱节点权重。

### 10.3 知识图谱模块 (核心演进)
构建 **五级分层知识体系**：
1. 技术领域 (如智能电网)
2. 岗位标准 (如巡检工程师能力模型)
3. 知识点 (如变压器原理)
4. 应用场景 (如发热故障排查)
5. 操作步骤 (实操流程)

### 10.4 学习路径推荐模块
- **以推理智能体为核心**
- 输入：个人图谱掌握度、学习进度、薄弱环节、目标岗位能力模型
- 输出：按个人节奏推进的定制化计划，改变“大班制被动听讲”。

### 10.5 产教融合生态对接模块
- 分析岗位能力与课程大纲缺口，反向支撑教学。
- 引入真实企业资源（现行标准、案例）。

### 10.6 深度问答 (GraphRAG)
基于多智能体和 Neo4j 图数据库，实现高难度的专业领域知识穿透。

### 10.7 学习行为反馈闭环
自动收集用户的学习动作（答题、视频停留等），交由系统演化个人图谱并触发路径自动纠偏。

## 11. 开发优先级
### 第一阶段：MVP基础
1. Neo4j 图数据库与五级图谱基础结构搭建
2. 用户登录、画像与初始个人图谱生成
3. 总控与检索智能体基础实现（GraphRAG）
4. 基础的推理学习路径生成

### 第二阶段：多智能体深度协同
5. 完善推理智能体逻辑与个人图谱动态演化
6. 产教融合缺口分析（教研支持）
7. 丰富的图谱前端可视化（AntV G6 / ECharts）

### 第三阶段：生态完善
8. 知识库更新智能体的闭环
9. 复杂的行为数据归因

## 12. 交互设计原则
- 首页核心突出：个人图谱当前状态、下一步定制计划、岗位能力差距。
- 图谱可视化作为强有力的支撑工具，允许用户自由探索“学了这个能干什么”。

## 13. 前端实现建议
- 图谱渲染必须考虑到大数量节点下的性能，优先展示关联子图。

## 14. 数据库实现约定
- 严格区分关系型数据与图数据。图谱相关（节点、拓扑关系、个人图谱边权重）强依赖 Neo4j。用户账号密码、基础字典放入 MySQL。

## 15. 风险提示
- 多智能体协同可能会增加系统延迟与 token 开销，需在设计时把控调用链条长度。
- GraphRAG 依赖于高质量的图谱构建，前期需要领域专家（教师/工程师）梳理导入扎实的“五级分层”数据底座。
- Cypher 语句的安全性与防注入需要重视。
