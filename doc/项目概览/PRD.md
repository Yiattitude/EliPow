# PRD

## 1. 背景与目标
针对电气相关专业“教材滞后、案例匮乏、学用脱节”的长期痛点，本产品旨在建立一个由**电力行业专属知识图谱驱动、多智能体协同**的个性化学习引擎。核心是将传统“一刀切”的被动听讲模式，转向以学生为中心、按个人能力图谱演化的主动学习模式，最终实现“在校所教所学即为企业所用”。

## 2. 目标用户
- 电气相关专业本科生
- 重点服务上海电力大学学生场景
- 高校教研团队（反向支撑教学大纲优化）

## 3. 产品目标
- **构建五级分层知识网络**（技术领域-岗位标准-知识点-应用场景-操作步骤）
- **动态演化个人知识图谱**，与通用图谱校准，精准识别知识缺口
- **多智能体协同**自动完成从学情识别到个性化学习方案生成的闭环
- **产教融合**：将企业真实运维案例与现行标准转化为教学化资源
- **反向赋能教学**：分析岗位能力与课程大纲缺口，辅助教师优化教学

## 4. MVP 范围
### 必做
1. 用户登录、画像采集与个人图谱初始化
2. 基于 Neo4j 的图谱存储与查询接口
3. **单智能体 GraphRAG 基础搭建 (V1 采用 System Prompt 代替多智能体编排)**
4. 基于图谱算法与规则的定制化学习路径生成
5. 产教融合资源关联展示与 GraphRAG 问答

### 暂不做或后期
- 社交与复杂社区
- 付费内容与课程交易
- 多角色复杂审批流程

## 5. 关键场景
- **路径生成**：新用户完成画像/测评 -> 推理智能体结合岗位标准与个人图谱 -> 生成高度定制化学习路径
- **知识演化**：学习行为追踪 -> 更新个人知识点掌握度权重 -> 图谱动态演化 -> 路径自动纠偏
- **深度问答**：用户提问 -> 检索智能体获取图谱上下文 -> 结合 GraphRAG 进行关联推理 -> 生成溯源回答
- **教研反哺**：系统汇总岗位能力标准 -> 对比学生当前能力图谱均值 -> 提示大纲缺口，反向支撑教师

## 6. 核心指标
- 个人图谱覆盖度与更新频次
- 路径推荐采纳率与完成率
- 产教资源库关联调用率
- GraphRAG 问答的推理准确性与溯源率

## 7. 交互原则
- 首页突出个性化学习路径、岗位目标差距与个人当前状态
- 强调知识图谱可视化（支持从宏观领域层层下钻到操作步骤）
- 资源与问答深度融合入学习节点

## 8. 风险与约束
- AI 输出需高度结构化、可解释，拒绝无图谱依据的自由生成
- 业务流程控制由多智能体框架的迭代校验控制，防范单点失效
- 复杂关联检索需保障图数据库（Neo4j）的查询性能
