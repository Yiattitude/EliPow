# 电荔 (Elipow) 系统全链路逻辑视图

本文档通过 Mermaid 流程图与序列图，将系统中所有核心功能的逻辑链条进行了可视化梳理，帮助您在编码前建立全局的流转思维。

## 1. 核心链路概览图 (Macroscopic Flow)

这是贯穿整个应用生命周期的主干流转逻辑：从新用户冷启动，到学习演化，再到最终的教研数据反哺。

```mermaid
graph TD
    %% 核心流程阶段
    A[注册/登录] --> B(首次登录画像采集)
    B --> C{是否完成初始诊断?}
    C -- 否 --> D[CAT 计算机自适应诊断测试]
    D --> E((生成初始图谱权重))
    C -- 是 --> E
    
    E --> F[推理智能体: 规划最短学习路径]
    F --> G[Dashboard 面板渲染 @xyflow]
    
    G --> H{用户交互}
    
    H -- "学习/测试" --> I[事件推送队列 RabbitMQ]
    I --> J[后台 Worker 计算 BKT 掌握度]
    J --> K[(更新 Neo4j 权重)]
    K -. WebSocket推送 .-> F
    
    H -- "自由提问" --> L[检索智能体 GraphRAG]
    L --> M[推理智能体 Socratic 引导]
    M --> N[溯源回答展示]
    
    K -. "T+1 跑批聚合" .-> O[(MySQL/Redis 宏观视图)]
    O --> P[教师端产教缺口沙盘]

    %% 样式
    classDef startEnd fill:#f9f,stroke:#333,stroke-width:2px;
    classDef aiAgent fill:#bbf,stroke:#333,stroke-width:2px,stroke-dasharray: 5 5;
    classDef db fill:#fdb,stroke:#333,stroke-width:2px;
    
    class A,P startEnd;
    class F,L,M aiAgent;
    class K,O db;
```

## 2. 局部关键逻辑拆解

### 2.1 冷启动：CAT 诊断测试链条 (Cold Start Diagnostic)

解决如何快速摸清学生底子，而又不引发用户疲劳的问题。

```mermaid
sequenceDiagram
    actor User as 学生
    participant UI as 前端 (React)
    participant C_Agent as 总控智能体
    participant Neo4j as Neo4j (行业图谱)
    
    User->>UI: 提交目标岗位 & 薄弱科目
    UI->>C_Agent: 发起冷启动 CAT 请求
    C_Agent->>Neo4j: 查询该岗位的 Hub 核心骨干节点
    Neo4j-->>C_Agent: 返回入度/出度最高节点列表
    
    loop 动态二分法测试 (至多15题/15分钟)
        C_Agent->>UI: 推送测试题 (针对当前 Hub 节点)
        User->>UI: 提交答案
        UI->>C_Agent: 答案评判
        alt 答对
            C_Agent->>C_Agent: 将前置节点标记为[掌握(跳过)]
            C_Agent->>C_Agent: 跃迁至更高难度节点
        else 答错
            C_Agent->>C_Agent: 向下钻取基础依赖节点
        end
        opt 触发熔断
            C_Agent->>C_Agent: 达到 15 题或 15 分钟 -> 强制 break
        end
    end
    
    C_Agent->>Neo4j: 初始化该用户的个人图谱 Mastery_Score
    UI->>User: 诊断完成，进入 Dashboard
```

### 2.2 多智能体学习路径生成 (Path Generation)

这是系统实现个性化、千人千面的核心环节。

```mermaid
flowchart LR
    Start([触发路径重算]) --> GetTarget[获取最终目标节点]
    GetTarget --> GetState[读取当前掌握度状态向量]
    GetState --> Search[图谱局部展开遍历]
    
    Search --> RuleCheck{是否满足护栏限制?}
    RuleCheck -- 跨度过大/超前学习 --> Fallback[强制回退至基础节点]
    RuleCheck -- 满足约束 --> Compute[计算认知负荷与最短路径]
    
    Compute --> AddBuffer[强制插入复习/缓冲节点]
    Fallback --> AddBuffer
    
    AddBuffer --> Render[返回 JSON 路径数据]
    Render --> UI[渲染 Dashboard]
```

### 2.3 渐进式苏格拉底 GraphRAG 链条 (Q&A Flow)

这是防范 AI 直接剧透、解决大模型幻觉的最硬核流程。

```mermaid
sequenceDiagram
    actor User as 学生
    participant System as 后端总控
    participant DB as Neo4j Vector Index
    participant Agent as 导师智能体 (Tutor Agent)
    
    User->>System: 提问："变压器为什么会短路？"
    System->>System: 提取关键词
    System->>DB: 发起 Hybrid Search (图圈定 + 向量召回)
    DB-->>System: 返回 1-3 个种子节点 & 文档 Chunk (带 ref_id)
    
    System->>Agent: 注入 Prompt (极严苛的 System Prompt + 上下文)
    Agent->>Agent: 单次 LLM 推理 (基于硬核 Prompt 强制进行苏格拉底式反问)
    Agent-->>User: 流式输出内容 (附带 `<ref>` 标签进行溯源高亮)
```

### 2.4 教研端双轨数据流演化 (Data Analytics Flow)

如何平衡图数据库性能与前端微观互动的实时性要求。

```mermaid
graph TD
    A[前端学习行为流水] --> B(消息队列 MQ)
    
    subgraph 微观实时流 (Event-Driven)
        B --> C{是否随堂互动场景?}
        C -- 是 --> D[Redis Pub/Sub]
        D --> E[Redis Hash 计数器 +1]
        E --> F[教师端毫秒级局部热力图刷新]
    end
    
    subgraph 宏观全局流 (T+1)
        B --> G[后台 Worker 异步消费]
        G --> H[(更新 Neo4j 个人图谱 Mastery_Score)]
        
        I((凌晨 Cron 定时跑批)) --> H
        H -. 读取并聚类 .-> J[全专业能力分布聚合计算]
        J --> K[(扁平化存入 MySQL/Redis)]
        K --> L[教师端产教缺口沙盘展示]
    end
```
