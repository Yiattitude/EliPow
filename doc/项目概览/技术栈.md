# 技术栈

## 概览
- 前端：React 生态 + UI 组件库 + 图谱复杂可视化
- 后端：Spring 生态 + MySQL/Redis + Neo4j (图数据库)
- AI：Spring AI + 专职多智能体编排 + Neo4j GraphRAG

## 核心技术栈

### 前端
- React
- Tailwind CSS
- shadcn/ui
- Lucide Icons
- React Router (路由)
- Zustand (全局状态管理)
- Axios (请求封装)
- React Hook Form + Zod (表单验证)
- react-markdown (渲染回答与资料)
- vite
- **图表与数据可视化**: ECharts (学习进度、画像能力分布雷达图)
- **图谱级可视化**: AntV G6 (支持大规模五级图谱及个人动态图谱的节点/关系交互展示)

### 后端
- Spring Boot
- Spring Security + JWT (认证鉴权)
- MySQL + MyBatis Plus (存储核心用户、系统配置、非关系型业务数据)
- Redis (缓存与会话)
- **图数据库**: Neo4j (存储“领域-岗位-知识点”五级知识网络、个人图谱关系权重、支撑 GraphRAG)
- Swagger / OpenAPI (接口文档)
- Jakarta Validation (参数校验)
- MinIO 或 OSS (文件存储)

### AI 与多智能体引擎
- **核心框架**: Spring AI (配合或补充 LangChain4j 等工具以实现多智能体编排)
- **多智能体架构**: 总控调度、知识检索、推理路径生成、知识库更新等多个专职 Agent 通信。
- **检索增强**: **Neo4j GraphRAG** (图检索增强生成，替代传统向量 RAG，执行深度的节点关联推理)。
- **主要能力**: ChatClient, Advisor, Tool Calling (供多智能体回调后端 API 更新图谱或查库)。

## 架构分工
- **Java 后端**：负责基础业务系统、用户认证、MySQL数据持久化、Neo4j图节点生命周期维护。
- **AI 多智能体引擎**：执行复杂推理逻辑（学情诊断、路径生成、图谱提取、产教融合缺口分析）。
- **前端**：负责高度定制化的学习路径渲染、宏观/微观结合的知识图谱交互探索。
