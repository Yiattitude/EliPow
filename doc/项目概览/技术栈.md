# 技术栈

## 概览
- 前端：React 生态 + UI 组件库 + 可视化
- 后端：Spring 生态 + MySQL/Redis
- AI：Spring AI + RAG + Tool Calling

## 核心技术栈

### 前端
- React
- Tailwind CSS
- shadcn/ui
- Lucide Icons
- React Router
- Zustand
- Axios
- React Hook Form
- Zod 
- ECharts
- AntV G6
- react-markdown 
- vite

#### 路由
- React Router 用于页面跳转、嵌套路由、权限路由控制。

#### 状态管理
- Zustand 轻量、易用，适合中小型项目的全局状态管理。

#### 请求封装
- Axios 用于统一 API 请求封装、拦截器、错误处理和鉴权处理。

#### 表单管理与校验
- React Hook Form
- Zod 用于登录、注册、画像填写、测评、后台管理等表单场景。

#### 图表与数据可视化
- ECharts用于展示学习进度、测评结果、成长曲线、能力分布等。

#### 知识图谱可视化
- AntV G6 用于展示知识点之间的依赖关系和学习路径结构。

#### Markdown 渲染
- react-markdown 用于问答内容、学习资料、知识说明等文本展示。

### 后端
- Spring Boot
- Spring Security
- MySQL
- Redis
- MyBatis Plus
- Swagger / OpenAPI
- Jakarta Validation
- JWT

#### ORM 框架
- MyBatis Plus 适合快速开发数据访问层，便于处理业务表和复杂查询。

#### 接口文档
- Swagger / OpenAPI 用于自动生成和维护接口文档，方便前后端联调。

#### 参数校验
- Jakarta Validation 用于接口入参合法性校验，提升系统稳定性。

#### 统一异常处理
- 全局异常处理机制用于统一返回错误码和错误信息。

#### 文件存储
- MinIO 或 OSS 用于存储课程资料、文档、图片、知识库文件等。

#### 定时任务
- Spring Scheduler / @Scheduled 用于资讯抓取、内容同步、推荐刷新等周期性任务。

#### 认证机制
- JWT 用于前后端分离场景下的登录态管理与鉴权。

#### 日志与审计
- 日志框架 + 请求日志用于跟踪系统调用、排查问题、分析 AI 结果。

### AI
- Spring AI
  - ChatClient
  - Advisor
  - PromptTemplate
  - Tool Calling / Function Calling
  - RAG (Retrieval-Augmented Generation)
  - Chat Memory / Conversation History

## 架构分工
- Java 后端：业务系统、用户系统、数据存储、权限、资源管理
- Spring AI 模块：问答、路径生成、资源推荐、资讯联动、工具调用
- 前端：交互、可视化、学习路径展示、图谱展示
