# 技术栈

## 概览
- 前端：React 生态 + UI 组件库 + 图谱复杂可视化
- 后端：Spring 生态 + MySQL/Redis + Neo4j (图数据库)
- AI：Spring AI + 专职多智能体编排 + Neo4j GraphRAG

## 核心技术栈

### 前端
- React
- Tailwind CSS
- shadcn/ui
- Lucide Icons
- React Router (路由)
- Zustand (全局状态管理)
- Axios (请求封装)
- React Hook Form + Zod (表单验证)
- react-markdown (渲染回答与资料)
- vite
- **图谱级可视化与选型考量 (混合策略)**: 
  - **局部路径/高交互图 (Dashboard 地铁图)**：采用 **`@xyflow/react` (React Flow 新版本)**。与 React 状态绝对契合，节点就是标准的 React 组件，完美适配 Tailwind CSS，杜绝了数据同步灾难，非常适合渲染包含复杂交互的“主干+支线”学习路径。需配合 `dagre` 或 `elkjs` 辅助自动布局。
  - **全局海量节点图 (图谱工作台)**：保留 **AntV G6 (WebGL)**。`@xyflow/react` 基于 DOM 渲染，超过 500 个节点性能就会面临挑战。因此面对动辄数千节点的“全局领域毛线球图谱”，依然必须依赖 G6 的 WebGL 渲染能力（严格采用增量更新防同步灾难）。

### 后端
- Spring Boot
- Spring Security + JWT (认证鉴权)
- MySQL + MyBatis Plus (存储核心用户、系统配置、非关系型业务数据)
- Redis (缓存与会话)
- **图数据库**: Neo4j (存储“领域-岗位-知识点”五级知识网络、个人图谱关系权重、支撑 GraphRAG)
- Swagger / OpenAPI (接口文档)
- Jakarta Validation (参数校验)
- MinIO 或 OSS (文件存储)

### AI 与多智能体引擎
- **核心框架**: Spring AI (搭配 LangChain4j 等工具实现单体 LLM 调度)
- **AI 架构 (V1降维策略)**: 废弃复杂的多智能体编排，全面采用 **“大单体 Agent + 极度严苛的防剧透 System Prompt”** 模式，实现伪苏格拉底式问答，极大降低延迟与工程复杂度。
- **检索增强**: **Neo4j GraphRAG** (图检索增强生成，替代传统向量 RAG，执行深度的节点关联推理)。
- **主要能力**: ChatClient, Advisor, Tool Calling (供多智能体回调后端 API 更新图谱或查库)。

## 架构分工
- **Java 后端**：负责基础业务系统、用户认证、MySQL数据持久化、Neo4j图节点生命周期维护。
- **AI 多智能体引擎**：执行复杂推理逻辑（学情诊断、路径生成、图谱提取、产教融合缺口分析）。
- **前端**：负责高度定制化的学习路径渲染、宏观/微观结合的知识图谱交互探索。
