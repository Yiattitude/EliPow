# 业务流程文档

## 1. 注册与图谱初始化
1. 用户注册/登录 (MySQL)
2. 采集初始画像与目标岗位 (MySQL)
3. 发起诊断性测评
4. 初始测评数据投递至 Neo4j，生成初始“个人知识图谱权重”

## 2. 测评与多智能体路径推荐
1. **总控调度智能体**：接收到用户路径生成请求。
2. **知识检索智能体**：从 Neo4j 提取该用户目标岗位的要求子图，及个人当前图谱子图。
3. **推理智能体**：综合对比上述子图，生成包含优先级的阶段性学习路径。
4. 路径展示至前端，引导用户按定制节奏推进。

## 3. GraphRAG 问答与资源关联
1. 用户提出专业疑问（例如针对某操作步骤）。
2. **总控智能体**：识别意图并下发检索指令。
3. **知识检索智能体**：在 Neo4j 中执行 Cypher 检索，将问题映射至具体知识节点，提取相关的依赖关系、前置知识、及挂载的资源（视频/文档）。
4. **推理智能体**：基于图谱上下文生成结构化回答，给出明确引用。

## 4. 产教融合资源入库流程
1. 企业文档/运维案例导入。
2. **知识库更新智能体**：提取文档中的操作步骤与应用场景。
3. 将提取的新节点自动挂载到 Neo4j 中现有的“技术领域/知识点”网络下。
4. （可选）触发重新评估受影响的岗位图谱模型。

## 5. 教学缺口反哺流程（教研端）
1. 系统后台汇总 Neo4j 中所有相关的最新岗位图谱（企业端）。
2. 对比高校现有教学大纲知识点。
3. 多智能体输出图谱缺口报告（如：“现行大纲缺失某某新能源并网实操场景”），提交教师端。

## 6. 学习进度闭环与图谱演化
1. 前端记录学习行为与测评结果流水。
2. 异步投递回业务系统，更新 Neo4j 中个人节点到知识节点的边权重（掌握度提升或下降）。
3. 当权重变化超过阈值，自动触发流程2，由推理智能体进行路径纠偏。
