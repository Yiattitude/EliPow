# 接口设计文档（API 文档）

## 1. 设计原则
- REST 风格
- 统一返回结构
- 明确分页与排序
- 关键接口支持审计

## 2. 核心接口概览
### 认证
- POST /api/auth/register
- POST /api/auth/login

### 用户画像
- GET /api/users/{id}
- PUT /api/users/{id}/profile

### 测评
- POST /api/assessments/start
- POST /api/assessments/{id}/submit
- GET /api/assessments/{id}/result

### 学习路径
- GET /api/learning-paths/current
- POST /api/learning-paths/recompute

### 资源
- GET /api/resources
- GET /api/resources/{id}

### 问答
- POST /api/qa/ask
- GET /api/qa/{id}

### 资讯
- GET /api/news
- GET /api/news/{id}

### 进度
- GET /api/progress
- POST /api/progress/update

## 3. 统一响应结构
```
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

## 4. 错误码约定
- 4001 参数校验失败
- 4010 未认证
- 4030 无权限
- 4040 资源不存在
- 5000 服务异常
