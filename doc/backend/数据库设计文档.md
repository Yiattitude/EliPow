# 数据库设计文档

## 1. 设计原则
- 采用 **关系型(MySQL) + 图数据库(Neo4j) + 对象存储(OSS)** 混合存储架构。
- MySQL 负责扁平化的核心业务数据存储（如账号、订单级记录等）。
- Neo4j 负责复杂关系网络存储，支撑 GraphRAG 极速穿透与智能体推理。
- OSS (如 MinIO/阿里云OSS) 负责多模态非结构化文件（PDF/视频/图片）的物理存储。

## 2. 存储选型划分

### 2.1 MySQL (关系型存储)
为符合《个人信息保护法》的最小必要原则，进行严格的物理/逻辑表隔离：
- **`auth_user` 表 (鉴权与账号基座)**：`id`, `encrypted_password`, `phone`, `wechat_openid`, `qq_openid`, `role`。处理多渠道绑定的账号合并逻辑。
- **`user_profile` 表 (学业与社交画像)**：`user_id`, `nickname`, `avatar`, `grade`, `future_plan`, `target_job`。
- **行为流水表**：用于大批量、结构统一的学习日志和事件埋点持久化。

### 2.2 Neo4j (图数据库存储)
为解决高并发与数据隔离问题，采用**读写分离与缓存架构**：
- **物理隔离策略**：使用 Neo4j Fabric 或多库实例模式，将“低频更新的全局行业图谱”与“高频更新的百万级个人动态图谱”物理分离，避免查询互相阻塞。
- **热数据内存驻留**：引入 Redis (或 RedisGraph) 缓存极高频访问的“专业基础子骨架”及“热门岗位标准图”，保障智能体毫秒级穿透速度。
- **联合检索 (Hybrid Search)**：利用 Neo4j 5.x 原生的 Vector Index，在同一条 Cypher 语句中融合“图遍历”与“向量相似度”（先圈定图谱范围，再做向量召回），极大降低 GraphRAG 幻觉。
  - *⚠️ 架构演进警告：当前采用 Neo4j 原生向量索引。若未来企业标准文档切片 (Chunk) 数量突破 500 万，需评估平滑迁移至 Milvus/Qdrant 的预案（系统需通过抽象 Repository 接口隔离底层的向量检索逻辑）。*

### 2.3 OSS 与多模态资源存储 (MinIO)
企业标准文档、教学视频、图片等大体积文件不入库，采用 MinIO/阿里云 OSS 进行存储。
MySQL 或 Neo4j 中的 `Resource` / `DocumentChunk` 节点必须包含以下核心字段：
- `oss_url`: 文件的物理存储路径。
- `file_hash`: MD5/SHA256 值，用于去重和防篡改。
- `mime_type`: 资源类型，用于前端决定渲染 PDF 预览器还是视频播放器。
- `chunk_text`: 仅存储纯文本切片，用于 Neo4j 向量索引和 LLM 上下文注入（绝不存储 Base64 或大段富文本）。

#### 图谱 Schema 定义 (预留文档扩展能力)
- **行业通用图谱 (全局只读/后台更新/五级网络)**：
  - [领域 Node] (如：智能电网)
  - [岗位 Node] (如：巡检工程师)
  - [知识点 Node]
  - [场景 Node] (真实企业案例映射)
  - [步骤 Node]
- **通用关系 (Edges)**：包含(HAS)、依赖(DEPENDS_ON)、前置(REQUIRES)、解决(SOLVES)等。
- **个人动态图谱 (数字孪生)**：
  - [用户 Node]：仅包含脱敏的 UUID (`userId`)，绝不包含社交隐私数据。
  - 用户的 [掌握关联 Edge] (例如 `(User)-[MASTERS]->(KnowledgePoint)`)
    - 边属性包含 BKT 算法所需核心字段：`mastery_score` (当前掌握概率), `prior_weight` (冷启动自述权重), `last_interact_at` (最后交互时间，用于遗忘曲线衰减), `interact_count` (真实交互次数)。
  - 用户的 [学习进度关联 Edge]

## 3. 关系概述 (以图为主)
- 岗位 -> 要求 -> 知识点集合
- 知识点 -> 构成 -> 场景/步骤
- 用户 -> (伴随时间轴演化的权重) -> 知识点
- 大纲体系 -> 映射对比 -> 岗位需求图谱 (用于教研反哺)

## 4. 约束
- 知识图谱节点的增删改查优先在 Neo4j 中保证事务性。
- GraphRAG 检索出的子图，必须提供明确的节点唯一标识，供智能体回溯上下文。
