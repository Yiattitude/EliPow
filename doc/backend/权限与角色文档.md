# 用户与权限管理设计文档

鉴于“电荔”平台引入了智能体引擎、知识图谱（Neo4j）以及产教融合（教研反哺）等复杂业务，用户管理模块不能仅仅局限于传统的“增删改查”，而是要实现**基础账号安全、多层级角色控制、跨双数据库映射（MySQL+Neo4j）、以及保护隐私的数据聚合**。

## 1. 核心角色与权限划分 (RBAC)

平台主要划分为三大核心角色群：

### 1.1 学生用户 (Student)
- **核心职能**：学习探索者，拥有专属的个人知识图谱。
- **权限范围**：
  - 渐进式画像采集与更新。
  - 获取由 AI 推理生成的个性化学习路径。
  - 调用 GraphRAG 沉浸式问答接口。
  - **AI 记忆管理**：允许重置自己的初始图谱测评数据（清除 AI 对自己水平的固有认知）。

### 1.2 教师/教研专家 (Teacher / Researcher)
- **核心职能**：教学优化者，利用系统的“教研反哺”能力。
- **权限范围**：
  - 查看“产教缺口沙盘”（对比高校大纲与企业岗标图谱）。
  - 查看基于全班/全专业学生图谱数据聚合生成的**群体能力热力图**。
  - 针对大面积薄弱知识点，推送集体学习任务。

### 1.3 系统管理员 (System Admin)
- **核心职能**：底层数据与生态的维护者。
- **权限范围**：
  - 审核与管理“知识库更新智能体”抓取的企业案例、标准文档。
  - 维护“五级分层知识网络”的底层拓扑结构（跨图谱的超级权限）。
  - 系统的基础运行监控与用户账号封禁/解冻。

## 2. 双数据库架构下的用户映射 (MySQL + Neo4j)

为了兼顾认证安全与 AI 图谱推理效率，用户的底层数据被严格拆分在两个数据库中同步：

### 2.1 MySQL 侧：基础账号与安全生命周期
- 存储用户的纯扁平化、敏感型安全数据。
- **核心字段**：`user_id`, `username`, `encrypted_password`, `wechat_openid`, `qq_openid`, `role`, `status`。
- **职责**：只负责鉴权、封号、密码重置、基础登录日志记录。绝不存储图谱关系。

### 2.2 Neo4j 侧：AI 图谱化身 (Digital Twin)
- 存储用户在 AI 引擎中的“数字孪生节点”。
- **节点定义**：`(:User {userId: "映射自MySQL", grade: "大三", futurePlan: "考研"})`
- **关系边定义**：通过引入前文提到的**贝叶斯知识追踪 (BKT)**理念，用户节点与知识节点之间建立动态边：
  - `(User)-[:MASTERS {weight: 0.85, last_interact: "2026-06-28", try_count: 5}]->(KnowledgePoint)`
- **职责**：仅供智能体引擎进行路径推演、GraphRAG 检索和相似度匹配，**不包含任何密码或鉴权敏感信息**。

## 3. 认证机制与渐进式冷启动 (Authentication & Cold Start)

### 3.1 统一认证网关 (Spring Security + JWT)
- 系统采用前后端分离架构，认证通过后下发 **JWT (JSON Web Token)**。
- 支持 OAuth2 协议接入微信和 QQ 的扫码登录，获取 OpenID 后在 MySQL 自动创建/绑定账号。

### 3.2 渐进式图谱初始化流程
配合前端的“先跳过，后采集”体验，后端的用户创建分为两个阶段：
1. **静默期**：扫码或注册瞬间，仅在 MySQL 创建基础账号，生成 JWT，允许用户进入系统“随便逛逛”。
2. **激活期**：当用户首次点击“生成我的路径”，或者提交了包含“薄弱项”的画像问卷时，系统触发异步事件。此时在 **Neo4j 中正式创建 `(:User)` 节点**，并根据问卷中勾选的“最头疼的课程”，初始化对相关知识点 `weight = 0.1` 的前置掌握度关系，完成冷启动。

---

## 4. 数据隐私与聚合规范 (隐私保护)

借鉴 MIT 平台保障隐私的做法，我们在为教师端（教研反哺页）提供数据时，必须遵守以下约定：
- **脱敏聚合**：教师端调用的分析接口，**不能拉取单个学生的具名图谱拓扑**。必须在后端通过 Neo4j 图查询，按“班级/专业”维度聚合数据（例如：查询某班所有 User 节点连接到“变压器发热”节点的平均 weight 值）。
- **防止算法茧房**：保证用户在请求重新评估（重置画像）时，后端的历史图谱关系能被真正归档或物理覆盖，防止 AI 永远受限于用户过去的错误表现。
