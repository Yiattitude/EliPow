# 异常与边界处理文档 (含大模型与图谱高可用规范)

鉴于“电荔”引入了单体智能体架构、MDP 路径生成与 Neo4j 图谱推理，本系统的异常处理重点不再是传统的 CRUD 校验，而是**AI 系统稳定性、幻觉控制与图谱数据安全**。

## 1. 单智能体大模型交互边界 (V1 降维版)
- **强 Prompt 约束 (防越狱/防剧透)**：由于 V1 废弃了多智能体互相校验的流程，系统完全依赖注入到单次 LLM 请求中的 System Prompt。必须引入强大的 Few-shot (少样本) 案例和反越狱指令，抵御用户试图骗取直接答案的行为。
- **可解释性与溯源约束**：LLM 输出流必须严格包含结构化的 `<ref id="xxx">` 标签。系统界面向用户展示内容时，必须基于图谱检索到的硬逻辑，禁止含糊其辞的“大模型自己计算得出”。

## 2. MDP 路径生成的安全护栏 (三阶段演进)
为防止状态爆炸和“刷奖励”行为，禁止一步到位上线纯在线强化学习。采用以下三阶段演进路线：
- **V1 (MVP)**：规则引擎 + LLM 辅助排序。固定图谱状态空间，验证基础可行性。
- **V2 (离线学习)**：使用专家教研数据训练策略网络（Imitation Learning），限制探索空间。
- **V3 (灰度在线)**：引入受控的 Online MDP。
- **防止探索伤害**：冷启动或低置信度时，Agent 禁止推荐跨度过大（>2个层级）的知识点，强制遵循“最近发展区”护栏。

## 3. 图谱更新与污染防护 (防投毒)
- **人类审核沙箱 (Human-in-the-loop)**：知识库更新 Agent 从企业标准中抽取的全新节点，**绝对禁止直接合入主图谱库**。必须先进入“待验证沙箱库 (Sandbox)”。
- **置信度分级**：只有经过多源交叉验证置信度极高（>0.95），或经教研专家人工点击“Approve”后，新节点才合并进生产级 Neo4j 图谱。
- **多源视冲突**：如果企业标准与教材知识发生冲突，系统在同一实体上挂载“企业视角”和“教材视角”双边属性，而非强制覆盖，交由前台根据用户的“未来规划（考研vs就业）”动态渲染。

## 4. Neo4j 图穿透性能熔断
- **最大跳数限制**：图穿透查询 (Traversal) 强制设置 `MAX_HOPS = 3`，防止底层引发全图扫描灾难。
- **混合检索架构**：放弃纯图遍历。采用 **GraphRAG 模式**：使用 **Neo4j 原生 Vector Index (联合检索)** 快速命中 1-3 个“种子节点 (Seed Nodes)”，再以此为圆心进行受限的局部子图展开（若未来 Chunk 突破 500 万，触发向 Milvus 迁移的预案）。
- **热点图谱物化**：对高频访问的“学科骨架图”，在 Redis 中进行物化视图缓存，防止高并发击穿图数据库。
