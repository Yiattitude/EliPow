# 知识图谱数据导入规范 (Data Ingestion Pipeline)

向系统导入“书本、教材、企业标准”等非结构化数据，并将其转化为 Neo4j 中的图谱节点与向量切片，是整个系统运作的基石。

对于《电荔》项目，我们采用**“物理存储分离 + LLM 结构化提取 + 混合图谱挂载”**的三阶段导入流。

---

## 阶段一：物理存储与版面解析 (物理层)

书本数据（如 PDF 格式的《电机学》）通常包含大量的排版、图片和表格，直接丢给大模型会引发严重幻觉。

1. **上传至 OSS/MinIO**
   - 教师/管理员在后台上传 PDF 教材。
   - 系统将物理文件存入 MinIO，并在 MySQL `RESOURCE_METADATA` 表中生成记录（包含 `oss_url`, `file_hash`）。

2. **文档版面分析 (Document Parsing)**
   - ❌ **错误做法**：直接使用 Python 的 `PyPDF2` 强行提取纯文本（会丢失层级和表格）。
   - ✅ **正确做法**：使用带视觉解析能力的光学引擎（如开源的 **Marker** 或 **Unstructured.io**）。
   - **目标**：将厚重的 PDF 转换为带有严格 Markdown 层级（`# 章`, `## 节`）和准确表格的纯文本格式，滤除页眉页脚噪音。

---

## 阶段二：智能分块与知识抽取 (知识层)

这是最核心的一步，将枯燥的文本变成图谱里的“知识点”。

1. **语义切片 (Semantic Chunking)**
   - 顺着 Markdown 的标题层级进行切片，确保每个 Chunk 包含一个完整的逻辑段落（约 500-800 Tokens）。
   - 每个切片生成一个 `DocumentChunk` 对象。

2. **LLM 辅助信息抽取 (Information Extraction Agent)**
   - 将上一步切好的 Chunk 喂给后台的“知识提取大模型”。
   - **Prompt 指令**：“你是一个电力专家。请阅读以下文本切片，提取其中的核心【知识点 (Knowledge Point)】，并找出这些知识点与已知的前置知识点之间的【依赖关系 (DEPENDS_ON)】。”
   - **JSON 输出示例**：
     ```json
     {
       "entities": [{"name": "变压器铁芯损耗", "type": "KnowledgePoint"}],
       "relationships": [{"source": "变压器铁芯损耗", "target": "电磁感应定律", "type": "DEPENDS_ON"}]
     }
     ```

3. **人类审核沙箱 (Human-in-the-loop)**
   - 提取出的节点和关系，先存入**沙箱区**。
   - 教研人员在后台界面进行快速点选确认（“是，变压器损耗确实依赖电磁感应”），防止 AI 瞎编乱造污染主图谱。

---

## 阶段三：入库与向量化 (存储层)

经过人工校验后，数据正式写入 Neo4j。

1. **写入图结构 (Cypher 注入)**
   - 创建知识点节点：`MERGE (k:KnowledgePoint {name: '变压器铁芯损耗'})`
   - 创建文档切片节点：`CREATE (c:DocumentChunk {chunk_id: '...', text: '...', source_pdf: 'oss_url...'})`
   - **建立支撑关系 (极其关键)**：`MERGE (k)-[:SUPPORTED_BY]->(c)`。这代表了这个知识点是由书本里的哪一段话作为权威解释支撑的。

2. **构建向量索引 (Vector Indexing)**
   - 调用嵌入模型 (Embedding Model, 如 `text-embedding-3-small` 或开源 `BGE-m3`) 将 `DocumentChunk` 的纯文本内容转化为高维向量。
   - 将向量更新到 Neo4j 节点中：`CALL db.create.setNodeVector(c, 'embedding', $vector)`。
   - 此时，Neo4j 的原生 Vector Index 自动生效。

---

## 💡 总结与调用示例

未来当学生提问“铁芯为什么发热”时，系统的 GraphRAG 流程如下：
1. 向量相似度命中书本里的**“铁芯损耗切片 (DocumentChunk)”**。
2. 顺着 `<-[:SUPPORTED_BY]-` 边，瞬间找到所属知识点**“变压器铁芯损耗”**。
3. 顺着 `-[DEPENDS_ON]->` 边，发现学生还没学过**“电磁感应定律”**。
4. AI 导师生成回答：“同学，在解释铁芯发热前，你还记得电磁感应定律吗？结合《电机学》第3章的内容（附带 PDF 原文高亮链接），思考一下交变磁场会产生什么？”
