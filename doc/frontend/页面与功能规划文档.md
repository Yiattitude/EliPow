# 前端页面与功能规划文档

## 2. 核心功能页面划分

### 2.1 登录与注册页
**功能目标**：完成账号体系打通与用户初始学情偏好数据的采集，需降低冷启动摩擦。
- **1.1 快捷登录**：支持账号密码登录，并**接入微信和 QQ 扫码**，降低使用门槛。
- **1.2 必填初始动态画像**：为了保障后续个人图谱和推荐算法的精准度，首次登录必须完成基础画像（不允许跳过）。采用分步卡片交互降低枯燥感，用户需选择：
  - 年级与未来规划（考研、考公、就业）
  - 目标岗位或目标院校
  - **当前最薄弱的专业课/知识点**（通过**标准课程库的下拉/搜索组件**进行选择，确保底层图谱映射的绝对准确，为 AI 提供绝佳的冷启动基准）。

### 2.2 仪表盘页 (Dashboard)
**功能目标**：用户每次登录后的学习主阵地，直观展示学习进度、提供实时行业动态，营造强烈的学习成就感。
- **2.1 布局原则**：
  - 上方：横向滚动式学习路径轴
  - 左下：岗位技能树点亮面板
  - 右下：实时信息咨询栏
- **2.2 “主干+支线”混合式路径轴 (Timeline)**：
  - **非线性依赖表达**：打破纯单线的视觉陷阱，采用“主干线+支线”或“换乘站”设计，真实展现多前置条件的“汇聚型依赖”。
  - **难度加权间距**：站点之间的物理间距采用“时间/难度加权”，并显式标注预估耗时，避免产生进度认知偏差。
  - **一句话推荐理由与校内映射**：悬浮节点时展示推荐理由（如“因上周错题频发触发复习”）及校内课程映射（“对应本校《电机学》第3章”）。
  - **每周时间预算与微调**：生成周计划前强制弹窗进行“每周时间预算确认”。允许用户拖拽微调节点顺序，此操作作为 RLHF 信号回传给算法。
- **2.3 岗位技能树点亮面板 (Skill Tree)**：摒弃传统雷达图，采用类似游戏设定的**“技能树点亮”或“核心能力充能进度条”**。面板内必须包含显眼的**“开始今日充能 (Start Learning)”**悬浮按钮，确保从“看”到“学”的转化链路少于 2 步。
- **2.4 实时信息咨询栏**：滚动展示。信息**非关联当前学习节点**，而是利用 AI 抓取并过滤与**用户最终目标岗位、行业前景**强相关的资讯（如最新考公/国网招考标准等），激发长线学习动力。

### 2.3 一站式沉浸知识图谱引擎 (Knowledge Graph Explorer)
**功能目标**：将“五级分层网络”以最直观的方式呈现。摒弃多页面跳转导致的上下文割裂，打造**一站式沉浸工作台 (Master-Detail View)**。
- **3.1 核心图谱主舞台**：全屏展示 2D/3D 拓扑网络。采用**按需加载 (Lazy Loading)**，仅渲染视口及 1-2 跳的节点。支持**语义缩放 (Semantic Zooming)**：缩小视角自动聚类为宏观领域簇，放大才展开具体节点，根治“毛线球”效应。
- **3.2 全局/个人视角无缝切换 (Toggle)**：左上角切换。切换时采用**平滑飞行 (Fly-to) 过渡 + 场景暗化**（非个人路径节点降低透明度），帮助用户维持绝对的空间方位感。
- **3.3 侧边透视面板 (Sidebar Drawer)**：在图谱中点击任意节点时右侧滑出。除资源外，支持**“What-If” 模拟预演**：用户可拖拽改变依赖关系或跳过节点，侧边栏实时警告认知断层风险（如“跳过此节点将导致后续理解率下降30%”）。

### 2.4 设置页
**功能目标**：标准化的用户个性化配置中心与 AI 数据管理。
- **4.1 基础配置**：界面风格/主题颜色选择，深浅模式切换，账号安全（绑定邮箱/手机号/第三方账号）。
- **4.2 个人画像管理**：允许用户在此处随时修改首次登录时填写的“未来规划、目标岗位、薄弱科目”等，修改后自动触发图谱权重的局部重算。
- **4.3 AI 记忆管理与重置**：展示“AI 目前认为我的水平/偏好”。提供**“我要重新测评”**或“清除特定学习记忆”功能，方便突击学习后的用户刷新初始图谱权重。

### 2.5 沉浸式问答页 (GraphRAG 面板)
**功能目标**：透明展示 AI 推理过程、提供精准溯源的极客级学习辅助工具。
- **5.1 悬浮/画中画模式 (PiP) 与上下文跟随**：问答窗口不仅支持独立全屏，还支持悬浮于图谱页。窗口顶部设有**“跟随图谱焦点”与“锁定当前上下文”的 Toggle 开关**，避免用户点击新节点时打断深度多轮对话。
- **5.2 核心模块组成**：
  - **对话主窗口 (Socratic UI)**：采用苏格拉底式引导，内置**“挫折感知降级机制”**（连续 3 次答错自动从“给提示”降级为“直接给公式/答案”）。增加**“一键转为闪卡 (Flashcard)”**功能，将顿悟时刻自动存入个人图谱。
  - **精准溯源侧栏**：后端 LLM 强制输出带有 `<ref id="node_x">` 的结构化标签，前端精准高亮侧边栏中的 PDF 切片（引入 Marker/Unstructured 解析），而非事后字符串盲猜。
  - **追问推荐 (Action Chips)**：结合图谱上下文预测的后续快捷提问标签。

### 2.6 教研反哺页 (教师/教研端)
**功能状态**：规划为**后续开发**。
- 用于提供大纲与企业需求的对比沙盘，以及全班学生群体能力热力图。

## 3. 前端工程与性能基准约束 (Tech Stack & Performance)
- **核心框架**：React + Vite
- **UI 与动效**：shadcn/ui + Tailwind CSS。
- **高性能可视化 (混合策略)**：
  - **全局复杂图谱**：必须使用支持 WebGL 的 **AntV G6** 或 Cytoscape.js，严禁在超 2000 节点时使用纯 Canvas/SVG。
  - **高交互局部路径图 (Dashboard)**：指定使用 **`@xyflow/react`** 处理包含复杂 React 组件的节点渲染，杜绝状态同步隐患。
  - **技能树**：避免高耗能的 3D 渲染，采用 **Lottie** 或 Spine 2D 序列帧动画保证低端设备流畅度。
- **数据刷新策略 (防白屏)**：Dashboard 作为高频聚合页，绝对禁止串行同步加载数据。必须采用**“骨架屏 + 异步分块加载 + WebSocket 局部推送”**机制。首屏秒开本地图谱骨架，资讯与 BKT 权重在后台计算完成后无感填入。
- **状态管理**：Zustand (管理当前图谱节点、用户会话状态等)。
